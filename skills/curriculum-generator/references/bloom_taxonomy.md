# ブルームのタクソノミー（改訂版）— Rise Path カリキュラム設計ガイド

> カリキュラム設計時に各モジュールの認知レベルを決定するために参照する。

## 6段階の認知レベル（低→高）

### 1. 記憶 (Remember)
- **行動**: 定義を思い出す、用語を識別する、事実を列挙する
- **動詞**: 定義する、列挙する、名前を挙げる、識別する
- **Rise Pathでの例**: 「Blenderのショートカットキーを3つ挙げてください」
- **モジュール設計**: フラッシュカード、用語集、○×クイズ

### 2. 理解 (Understand)
- **行動**: 概念を説明する、要約する、言い換える
- **動詞**: 説明する、要約する、分類する、比較する
- **Rise Pathでの例**: 「メッシュとカーブの違いを自分の言葉で説明してください」
- **モジュール設計**: whyItMatters セクション、analogy セクション

### 3. 応用 (Apply)
- **行動**: 知識を新しい状況に適用する、手順を実行する
- **動詞**: 実行する、使用する、解決する、実装する
- **Rise Pathでの例**: 「この手順でUV展開を実際にやってみましょう」
- **モジュール設計**: actionStep セクション、ハンズオン演習

### 4. 分析 (Analyze)
- **行動**: 情報を部分に分け、関係性を見つける
- **動詞**: 比較する、対比する、区別する、原因を特定する
- **Rise Pathでの例**: 「このコードがエラーになる原因を分析してください」
- **モジュール設計**: デバッグ課題、ケーススタディ

### 5. 評価 (Evaluate)
- **行動**: 基準に基づいて判断する、正当化する
- **動詞**: 判断する、批評する、正当化する、推奨する
- **Rise Pathでの例**: 「2つのモデリング手法のどちらが適切か、理由と共に選んでください」
- **モジュール設計**: 比較課題、コードレビュー演習

### 6. 創造 (Create)
- **行動**: 新しいものを設計する、企画する、制作する
- **動詞**: 設計する、制作する、構成する、発明する
- **Rise Pathでの例**: 「自分のオリジナルキャラクターを1体モデリングしてください」
- **モジュール設計**: プロジェクト課題、ポートフォリオ作成

## カリキュラムへの適用ルール

| モジュール位置 | 推奨レベル | 理由 |
|:---|:---|:---|
| 最初の1-2モジュール | 記憶 + 理解 | 基礎固め、自信の構築 |
| 中盤 | 応用 + 分析 | 実践力の養成 |
| 終盤 | 評価 + 創造 | 統合的な学びの実現 |

## Big5との組み合わせ

- **誠実性 低**: 早期に「応用」レベルを入れる（体験先行）
- **開放性 高**: 「創造」レベルの課題を多めに
- **神経症 高**: 各レベルで安全な失敗環境を提供
